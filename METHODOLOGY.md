# pptdog 方法论提炼（内部参考，供各 SKILL.md 引用）

## 一、分享/演讲方法论（来源：听得进/记得住/有启发）

### 第一层：听得进
**核心公式：未知 → 有价值 → 少即是多**

- **空姐效应**：听众已知的内容，再重要都没用。两种已知来源：
  1. 听众早就知道了
  2. PPT 上写了，你再读 PPT（眼睛盯着 PPT 就离不开了）
- **听众是谁**：三条调研路径（调研报告 / 问出品人 / 猜历史安排）
- **已知地图**：听众通常知道 What 和 How，但不知道 Why（特别是更深入的 Why）
- **内容定位**：每个内容块必须有存在理由（架构图要有红框指引，段子要有逻辑功能）
- **断舍离**：有限时间讲深讲透 > 面面俱到；多即是少，克制才是爱

### 第二层：记得住
**核心公式：有关联 → 有脉络 → 有锚点**

- **大脑记忆原理**：有关联就好记（类比 LLM 的 token 关联）
- **金字塔原理**：结论先行，论据支撑，事实验证
- **思维导图先行**：绝对要思维导图先行，再写内容
- **一波三折**：故事脉络，用于分享型（见下方模板）
- **逻辑连接词**：演讲时强调"更进一步看""更深入的原因""其他可能性"
- **记忆锚点**：
  - 开门：好故事 / 问题引导 / 反常识
  - 关门：重复 / 升华 / 流程 / 地图
  - 标题 = 论点（不是话题）

**一波三折提示词模板：**
```
挑战和失败：[产生共鸣的情景和细节，听众在现场的模样，自己的感受]
思考：[至少3个根因分析：原因 / 更深层原因 / 更更深层原因，附推导逻辑和解决方案]
成功的结果：[量化数字 + 共鸣的情景细节]
新挑战（递进，更难）：[同样要有情景和细节]
再次思考：[与上次有递进关系，更深层更细节更本质]
成功的结果：[...]
最高潮挑战：[...]
再次思考：[...]
最终成功：[...]
```

**大纲标题优化提示词：**
```
结合上面一波三折的故事情节输出分享大纲和内容简介，要求：
- 与听众痛点相结合
- 每个部分的小标题都要有吸引力
- 多用数字，多戳痛点
- 控制在 21 个字之内
```

### 第三层：有启发
**核心公式：贴近 + 深度 + 可复用**

- **距离问题**：高屋建瓴需要桥梁；每个 PPT 都要有工作中的真实案例
- **深度思考工具**：
  - F.A.B 法则：Feature → Advantage → Benefit（采集→结构化→剔已知→突重点）
  - 五个为什么
  - MECE 原则
  - 象限图 / 二元对立→整体论
  - AI 辅助深化提示词（问题→原因→更深层原因→方法→进一步思考）
- **授人以渔**：分享本质是有压力的深度思考，也是验证思想的机会

## 二、汇报方法论（来源：汇报应该怎么说）

**本质：通过举证成果，向评委/听众证明价值并让其相信的过程**
（四关键词：成果 / 自己 / 牛 / 评委）

### 成果举证
**成果 ≠ 事情**
- ❌ 建设质量体系
- ✅ 通过构建质量保障体系，XX应用持续保持零故障单

**有价值的成果**：基于业务/用户痛点的思考，痛点越痛，成果越有价值
- 花 1-2 页证明"这个痛有多痛"才能衬托成果价值
- 反模式："复杂痛"（放架构图说自己很复杂）→ 无效
- 正确做法：数字化痛点（3个9的服务质量 = 每月损失超过1000万）

**从不痛到很痛的陈述梯度：**
```
不痛：我们是个web服务，需要提升SLO
一般：我们是个核心web服务，涉及业务关键流程
较痛：web服务涉及客户支付流程
很痛：三个9的服务质量 = 每月损失超过1000万
```

**真实的成果**：夸张数据要有证据（监控截图 + 人证/客户认可截图）

**可复制的成果**：有方法论 = 可复制；方法论在多项目落地 = 最强证据

### 证明"自己"
- 用"我"不用"我们"（"我们"=团队成果）
- 多人合作时：架构图圈定边界，说清自己的贡献度和因果关系
- 技术难点的思考过程是核心举证
- 心态：无欲则刚，当评委是来学习的人

### 展现"牛"

**❌ 运气牛反模式**：把因缘际会（如疫情流量）说成推动力，显得是碰运气
**✅ 正确做法**：还原当时的思考，体系化、因果关联，证明是我主动抓住机会

**技术牛：站在巨人肩膀**
- 找巨人：Google Scholar / DroidCon / QCon / CNCF / Databricks 等开发者峰会
- 欣赏巨人，寻找启发（看优势，不只看不足）
- 使用方式：直接使用 / 升维使用 / 组合使用 / 修改使用 / 思路借用
- 竞品对比：站在巨人肩膀，不是踩在矮人身上（"防御性竞品"≠技术牛）

**难题有多难**：
- 竞品落地效果对比，突显突破难度
- 递进式描述：通常做到A/B/C，我们还做了D，甚至E

**思路牛：体系化思考**
拆解问题方式（代表格局）：
1. **参考经典**：韧性架构 / 分层 / 发现-分析-解决-度量 / 预防-检测-缓解-修复 / 戴明环
2. **自行归纳**：从历史事故根因总结，注意：从原因出发（非现象），符合MECE
3. **1+2结合**：经典框架代入现状，看遗漏，针对某点进一步拆解

分析过程：结合情景说明为何选此拆解方式（时间紧/长期问题/组织割裂 → 不同最优解）

解决方案：短期+长期配合（近交远攻）；遗留问题+未来规划给评委"气口"

### 常见问题（临阵磨枪）

**假问题识别**：出现"不/没有/非"描述问题 → 说的是解决方案
```
❌ 假问题：K8S的资源无法复用，导致无法支持更多业务
✅ 真问题：成本控制之下，资源有限但业务增长快速
```

**推导过程缺失**：为什么这样做 > 做了什么；解决方案单一 = 没有推导过程

**标题废话**：
```
❌ 发现了一个问题（什么问题？）
❌ K8S不满足我们的诉求（什么诉求？）
❌ 通过XX策略，突出优势（什么优势？）
```
电梯法则：标题=论点，快速传递关键信息

## 三、思维导图方法论（来源：两份素材综合）

### 思维导图的核心价值
- 写得快（30分钟完成汇报PPT骨架）
- 看大局（左右脑切换）
- 记得住（自己和听众都好记）
- 动态过程，工具辅助（mindnode / processon），开始乱七八糟是对的

### MECE 原则
- 无重复、无遗漏
- 有价值的分类（时序/输入/网络/磁盘 > 男/女这种废分类）
- 补充同维度：有内存 → 补CPU/网络/磁盘/GPU
- 归纳抽象：有男人/女人 → 抽象出"性别"维度
- 横向平推：同一分类方式能否应用到其他节点

### 因果推演（一波三折）
- 问题 → 解决方法 → 结果，层层递进
- 至少3层根因：原因 / 更深层原因 / 更更深层原因

### 子节点深化固定属性
想不到子节点内容时，用这些框架启发：
- 优缺点
- 5W2H（Why/What/Who/When/Where/How/How much）
- 相似/相反
- 宏观/微观，主观/客观，内/外
- 四因论（质料因/形式因/动力因/目的因）

### 思维导图转 PPT
- 开门：描述背景，引导听众主动思考（好故事/问题/反常识）
- 关门：小关门回顾核心，大关门升华主题价值
- F.A.B 步骤：采集素材 → 结构化 → 剔除已知 → 突出有利信息
- PPT 上不写你要读的内容（空姐效应）
