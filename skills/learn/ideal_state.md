# 理想态：learn

## 概述

`/learn` 是 pptdog 的经验库管理入口，负责让讲者随时能"翻看自己积累了什么"。它的价值不在于对话引导，而在于信息呈现的准确性与数据操作的安全性。每次打开时，AI 应立即从 `~/.pptdog/learnings.jsonl` 读取真实数据，计算统计摘要，并以分层清晰的格式展示，让讲者一眼就能判断经验库的健康状况。

Preamble 统计是本 Skill 的"门面"——总条数、类型分布、平均置信度、最近 7 天新增，这四项数字必须来自实际文件计算，不能虚构或估算。置信度显示需应用衰减规则（`observed/inferred` 每 30 天 -1），而不是直接读取原始 `confidence` 字段。统计摘要的最近 7 天条目需要逐条列出，让讲者能立刻回忆起最新发生的事。

`/learn` 的核心风险来自两处：一是**数据破坏**（修剪/删除时未二次确认，或误删 `user-stated` 记录）；二是**引导缺失**（记录为空时没有解释经验如何积累，用户不知道下一步）。理想态要求所有写操作前必须展示待操作内容并等待确认，`user-stated` 条目在修剪时始终受保护，空库时给出清晰的积累时机说明和手动添加入口。

## 输出规格

### Preamble 统计块（每次启动必显示）

必须包含以下四项，缺一不可：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  🐕 pptdog learnings 管理
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  共 N 条记录
  pitfall: X  |  insight: X  |  correction: X  |  preference: X
  平均置信度：X.X/10

  最近 7 天新增：N 条
  • [skill名称 / type] key值  (n/10) — YYYY-MM-DD
  • ...
```

**计算规则：**
- 总条数：JSONL 文件有效行数（跳过空行和解析失败行）
- 类型分布：按 `type` 字段统计，四类均需显示（即使为 0）
- 平均置信度：应用衰减规则后的当前有效置信度均值，保留一位小数；过滤掉 `confidence < 3` 的条目后计算
- 最近 7 天：以当前时间为基准，`ts` 字段在 7 天内的条目；逐条列出，按时间倒序

### 空库时的提示块

```
  暂无 learnings 记录。

  pptdog 会在以下时机自动积累经验：
  • /ppt-review 发现问题时（写 pitfall）
  • 用户修改 AI 建议时（写 correction）
  • 用户明确表达偏好时（写 preference）
  • 某个环节特别顺利时（写 insight）

  你也可以选择 [G] 手动添加经验。
```

### 功能菜单（Ql-1）

Preamble 之后立即展示，等待用户选择：

```
你想对 learnings 做什么？

A. 浏览全部（分页）                ← 推荐（第一次进来先看看积累了什么）
B. 按 skill 过滤
C. 按场景过滤（分享/汇报）
D. 搜索关键词
E. 修剪低置信度条目（< 5分）
F. 删除某条记录
G. 手动添加经验
H. 导出为 Markdown 报告
0. 退出

💡 第一次使用建议选 A 浏览全貌；有具体关键词时选 D 最快；想清理数据质量时选 E。
```

### 浏览格式（[A] 及其他过滤功能复用）

每条条目格式：
```
#N  [skill名称 / type]  key值
    ★★★★★★★★☆☆  N/10
    insight 文本
    来源: observed/inferred/user-stated  |  场景: context值  |  时间: YYYY-MM-DD

```

- 置信度星号：N 颗实心 ★ + (10-N) 颗空心 ☆，共 10 颗
- 每条之间空一行
- 页尾：`[n] 下一页  [p] 上一页  [m] 返回菜单`

### 修剪确认块（[E]）

```
以下条目置信度 < 5 分，建议删除（共 X 条）：

  #1  [skill / type]  key  (N/10)
      insight 摘要
      来源: inferred  |  时间: YYYY-MM-DD

  ...

注意：source=user-stated 的条目已跳过（共 Y 条受保护）
```

确认菜单（Ql-3）：
```
确认删除以上 X 条低置信度记录？

A. 确认删除（不可恢复）
B. 取消，返回菜单            ← 推荐（默认安全操作）

💡 被保护的 user-stated 条目不会被删除；如需删除用户添加的条目，请用 [F] 逐条操作。
```

### 手动添加引导（[G]，6步）

Step 1（Ql-5）：类型选择（A/B/C/D）  
Step 2（Ql-6）：关联 skill（A-F + 0.不确定）  
Step 3（Ql-7）：场景（A/B/C）  
Step 4：key 标识 + 一句话描述（两个输入）  
Step 5（Ql-8）：置信度（A/B/C/D + E.自定义）  
Step 6（Ql-9）：预览 JSON + 确认写入（A/B/C）

写入成功输出：
```
✓ 已记录！随着使用次数增加，pptdog 会越来越了解你的演讲习惯。
```

## 质量标准

### 统计准确性

**高质量表现：**
- 四类类型（pitfall/insight/correction/preference）均显示，即使某类为 0
- 平均置信度计算时应用了衰减规则，不是直接读取原始字段
- 最近 7 天新增逐条列出，包含 skill、type、key、当前置信度、日期
- 文件不存在时优雅降级为空库提示，而非报错

**低质量表现：**
- 类型分布漏掉了值为 0 的类型
- 平均置信度直接使用原始字段未经衰减计算
- 最近 7 天只显示条数，不显示具体条目

### 数据安全性

**高质量表现：**
- 修剪前列出所有待删除条目，并明确标注 user-stated 受保护数量
- 删除单条前显示条目详情，二次确认后才执行
- 写操作默认选项均为"取消/返回"，安全第一
- 修剪和删除操作后，返回菜单并可选择继续操作

**低质量表现：**
- 修剪时未展示待删除列表就直接删除
- user-stated 条目参与了自动修剪
- 删除确认只用一句话，未展示条目详情

### 手动添加引导质量

**高质量表现：**
- 每步只问一个问题，提供选项而非纯开放输入
- Step 4 分两步：先收 key（简短标识），再收 insight（一句话描述）
- Step 6 展示完整 JSON 预览，讲者可以核实每个字段
- 置信度选项给出了具体的置信区间说明（如"亲身验证多次"）

**低质量表现：**
- 一次性要求用户填写所有字段
- 没有 JSON 预览步骤
- 置信度只让用户输入数字，没有区间说明帮助判断

## 失败模式

### F1：统计数字失真

Preamble 中的统计值与实际文件内容不符——条数少算、平均值未应用衰减、最近7天时间窗口计算错误。

**识别信号：** 统计数字与文件实际内容有偏差；或 `confidence < 3` 的衰减条目被纳入平均计算。

---

### F2：修剪误删 user-stated

执行 [E] 修剪时，`source == "user-stated"` 的条目被列入待删除清单，甚至被删除。

**识别信号：** 修剪预览列表中出现 `来源: user-stated` 的条目；或修剪后发现 user-stated 条目数量减少。

---

### F3：空库无引导

`learnings.jsonl` 为空或不存在时，AI 只说"暂无记录"然后显示菜单，没有解释经验如何积累，用户不知道下一步该做什么。

**识别信号：** 空库提示缺少"pptdog 会在以下时机自动积累经验"的说明；或没有引导用户到 [G] 手动添加。

---

### F4：删除无二次确认

[F] 删除某条记录时，AI 未展示条目详情就直接删除；或确认步骤中默认选项为"确认删除"而非"取消"。

**识别信号：** 删除操作只有一次确认；或用户输入序号后立即删除，没有展示条目内容。

---

### F5：手动添加引导断裂

[G] 手动添加过程中，某步骤的输入没有被正确传递到下一步；或最终 JSON 预览中字段值与用户输入不一致；或写入后没有成功确认提示。

**识别信号：** Step 6 预览 JSON 中出现占位符或空字段；写入后没有 `✓ 已记录` 的确认；或整个引导过程少于 6 步就结束。

## 评分维度

以下维度用于 Judge 对 `/learn` 的 Preamble 输出和操作响应进行评估，每个维度 0-10 分。

---

### D1：Preamble 统计准确性（0-10）

**评估：** 统计摘要中的四项数字是否来自正确计算，且格式完整。

| 分数段 | 描述 |
|--------|------|
| 8-10 | 四项均正确（总条数/类型分布/衰减后平均置信度/最近7天逐条列出），格式符合规格 |
| 5-7 | 主要信息正确，但部分格式不规范或缺少最近7天逐条展示 |
| 2-4 | 统计有明显错误（如类型缺失、未应用衰减规则）|
| 0-1 | 统计数字严重失真，或完全省略 Preamble |

---

### D2：菜单完整性（0-10）

**评估：** 功能菜单是否包含全部 9 个选项（A-H + 0），且提示语准确。

| 分数段 | 描述 |
|--------|------|
| 8-10 | 9 个选项全部存在，推荐标注准确，使用提示语贴切 |
| 5-7 | 选项基本完整，但推荐标注或提示语有偏差 |
| 2-4 | 缺少 1-2 个选项，或推荐标注混乱 |
| 0-1 | 菜单结构严重缺失 |

---

### D3：浏览格式规范性（0-10）

**评估：** [A] 浏览输出的每条条目是否符合格式规格（星号置信度/分页/导航）。

| 分数段 | 描述 |
|--------|------|
| 8-10 | 星号数量准确（N颗实心+10-N空心），每页10条，页尾导航完整 |
| 5-7 | 格式基本正确，但星号数量偶有偏差或缺少分页导航 |
| 2-4 | 置信度只显示数字未显示星号，或无分页 |
| 0-1 | 格式混乱，无法识别条目边界 |

---

### D4：修剪安全性（0-10）

**评估：** [E] 修剪流程是否正确保护 user-stated 条目，且有完整的确认机制。

| 分数段 | 描述 |
|--------|------|
| 8-10 | user-stated 完全跳过（并标注保护数量），待删列表完整展示，默认选项为"取消" |
| 5-7 | user-stated 保护正确，但确认机制或默认选项有问题 |
| 2-4 | user-stated 部分参与了修剪，或未展示待删列表 |
| 0-1 | 直接删除，无确认，或 user-stated 被删除 |

---

### D5：手动添加引导完整性（0-10）

**评估：** [G] 的 6 步引导是否完整，每步是否提供选项，JSON 预览是否准确。

| 分数段 | 描述 |
|--------|------|
| 8-10 | 6 步完整，每步有选项，Step 6 JSON 预览字段与用户输入完全一致，写入有确认 |
| 5-7 | 步骤基本完整，但个别步骤缺少选项或 JSON 预览有小偏差 |
| 2-4 | 步骤不足 5 步，或 JSON 预览缺失 |
| 0-1 | 引导中断或写入无确认 |

---

### D6：空库处理质量（0-10）

**评估：** 当 learnings.jsonl 为空或不存在时，AI 的响应是否给用户足够的方向感。

| 分数段 | 描述 |
|--------|------|
| 8-10 | 明确说明"暂无记录"，列出 4 种自动积累时机，并引导到 [G] 手动添加，菜单完整展示 |
| 5-7 | 有空库说明，但积累时机说明不完整或 [G] 入口不明显 |
| 2-4 | 只说"暂无记录"，无积累说明 |
| 0-1 | 报错或直接崩溃，无降级处理 |
