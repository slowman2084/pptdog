# 理想态：review-structure

## 概述

`review-structure` 是 pptdog 审查链路中专注于**叙事弧度与开关门质量**的审查器。它的审查对象是**演讲的框架和节奏**，而不是单页内容是否充分或论证是否严密——后者分别由 `review-content` 和 `review-logic` 负责。

该 Skill 从整体结构视角审查九个维度：大开门/大关门力度（首尾钩子）、章节小开门/小关门（每章进出仪式感）、一波三折（分享型叙事弧）或金字塔原理（汇报型结论先行）、逻辑连接词（章节间不跳跃）、Slide 数量与时长匹配、思维模型识别（增益补全）。

---

## 需求体系

### 固有需求分类（视角 A：叙事完整性层次）

| ID | 固有需求 | 定义 |
|----|---------|------|
| I1 | 开关门力度 | 大开门在前30秒抓住听众，大关门留下可复述的核心结论 |
| I2 | 章节钩子 | 每章有进入感（小开门制造期待）和落地感（小关门固化结论）|
| I3 | 叙事弧度 | 分享型有情绪递进（一波三折），汇报型有结论先行（金字塔）|
| I4 | 连接流畅 | 章节间有逻辑连接词，听众不感觉跳跃 |
| I5 | 思维模型 | 已有框架接近某个强模型时主动补全缺失维度（锦上添花）|

### 主需与次需

**主要需求（决定审查结果核心价值）：**

- **I1 开关门力度** → 落地为：大开门检测有效钩子类型（好故事/反常识/问题引导）；大关门检测升华+行动号召+可复述金句；severity 分级严格（废话开场 = critical）
- **I2 章节钩子** → 落地为：每章小开门必须基于本章真实素材；小关门必须固化核心结论并有口语化过渡语；缺失或废话过渡均触发 suggestion

**次要需求：**

- **I3 叙事弧度** → 以 ppt-hours.md 的分享类型字段为准（D2A）；分享型检查一波三折，汇报型检查金字塔；类型字段缺失时 fallback 到"询问用户"而非推断
- **I4 连接流畅** → 章节过渡语扫描有效逻辑连接词（递进/因果/转折/总分）；"第X章"机械切换触发 suggestion
- **I5 思维模型** → 严格门槛：匹配度 ≥80 才输出（D1A），最多 3 条；低于阈值不生成，不强行识别

### 需求间关系

```
I1(开关门力度) ──┐
                 ├── 共同决定听众能否被带入并带走演讲的核心价值
I2(章节钩子) ───┘

I3(叙事弧度) ←── I1+I2 的整体骨架：首尾和各章节有力度，整体才有弧度
I4(连接流畅) ←── 微观层的胶水：强大的钩子之间需要自然的过渡
I5(思维模型) ←── 超越现有内容的增益：在既有结构上提供框架升级机会
```

## 质量标准

### Preamble 执行

- 版本检查 + learnings 搜索 + slug 识别（标准逻辑）
- 读取 slide-content.md（不存在时停止）
- 读取 ppt-hours.md 获取**分享类型**和目标时长（缺失时标注"未知"）
- 打印文件统计（行数/页数/章节数/分享类型/目标时长）

### 分享类型判断（D2A — 只看 ppt-hours）

- 从 ppt-hours.md 的类型字段读取（分享型 / 汇报型 / 混合型）
- **类型存在**：分享型→执行维度5，汇报型→执行维度6，混合型→两个都执行
- **类型缺失/无法读取**：在状态摘要中标注"分享类型未知"，**询问用户** `「请告诉我这是分享型（故事叙事）还是汇报型（结论先行）？」` 后再继续——**不推断**
- 不依靠 slide-content.md 结构特征来推断类型

### 维度 1：大开门力度

触发条件（满足任意一条）：
- 开场是自我介绍（"大家好，我是..."）
- 开场是目录展示（无钩子直接列议程）
- 开场是背景介绍（平铺直叙无冲突/悬念）
- 有钩子但内容空洞（"大家有没有遇到过挑战？"← 太宽泛，听众感知不到）

有效钩子类型（不触发）：好故事（有场景/冲突/人物）/ 反常识（颠覆预期的数字）/ 问题引导（直指听众痛点的具体提问）

severity：`critical`=废话开场 / `major`=有钩子但空洞 / `minor`=钩子存在但画面感不足

### 维度 2：大关门力度

触发条件：
- 以"谢谢大家"/"有问题欢迎提问"结束
- 结尾只是重复前面内容，无升华
- 缺少行动号召或可复述的标志性结论

severity：`critical`=无实质结尾 / `major`=内容重复无升华 / `minor`=有升华但过于抽象难复述

### 维度 3&4：章节小开门/小关门

**小开门触发条件**：
- 章节开头是"接下来我们看第X章"/"下面进入..."
- 小开门内容与本章素材无关（泛化问句）
- 完全缺少小开门直接从论点开始

**小关门触发条件**：
- 章节结尾无实质总结直接跳下一章
- 结论模糊（"总结一下，我们做了很多工作"）
- 过渡语是书面语机械切换

severity：`critical`=超过半数章节缺失 / `major`=某重要章节缺失或废话 / `minor`=存在但内容与素材脱节/不够口语化

### 维度 5：一波三折完整性（仅分享型）

检查整体叙事有无递进挑战结构（至少两折）：
- ≥2 折（有递进）：不触发
- 1 折（单次挑战→成功，无新挑战）：`major`
- 0 折（线性列举 A→B→C 无故事弧）：`critical`

### 维度 6：金字塔原理（仅汇报型）

检查：
- 结论是否先行（第一张实质内容 Slide 即点核心结论）
- 论据是否平行不重叠
- 每个论据是否有数据支撑

severity：`critical`=结论在末尾（金字塔倒置）/ `major`=结论先行但论据有逻辑跳跃 / `minor`=结构正确但某论据缺数据

### 维度 7：逻辑连接词

扫描所有章节间过渡文本，检测是否使用有效逻辑连接词：

有效类型：递进（更进一步/更深层是...）/ 因果（正是因此/这导致了...）/ 转折（但我们发现/看似解决了实则...）/ 总分（概括来说/这里有N个要素）

触发：`"接下来第X章"` / `"好，说完了，下面"` / `"进入下一个话题"` 等机械切换

severity：`critical`=所有过渡都是机械切换 / `major`=超过半数缺逻辑连接词 / `minor`=个别过渡生硬

### 维度 8：Slide 数量与时长匹配（D3B — 报告+具体建议）

**公式**：目标时长(分钟) × 0.6 ≤ 合理 Slide 数 ≤ 目标时长(分钟) × 1.2

计算实际 Slide 数，对比合理范围：

- 在范围内：不生成 suggestion
- 偏少（< 目标×0.6）：生成 suggestion，**具体指出哪些论点可能需要拆分为 2 页**（基于内容密度判断）
- 偏多（> 目标×1.2）：生成 suggestion，**具体指出哪几页内容较简单可合并**

severity：`critical`=偏差>50% / `major`=偏差20-50% / `minor`=偏差10-20%

### 维度 9：思维模型识别（D1A — 严格门槛 ≥80）

**识别流程**：
1. 扫描 mindmap.md 和 slide-content.md 中同层级并排节点
2. 对每组评估与已知思维模型的匹配度（0-100）
3. **≥80 分**：生成 suggestion，type=`"opportunity"`
4. **50-79 分**：不生成
5. **< 50 分**：不生成

每次审查最多输出 3 条思维模型 suggestion；若所有组匹配度 <80，此维度输出 0 条（正常）。

suggestion 必须包含：model_name / model_domain / match_score / 缺失维度说明 / 3 个有实质差异的补全选项（content 写具体内容，不是方向说明）。

### 建议数量

- 期望范围：内容 ≤ 20 页→5-10 条，> 20 页→8-15 条
- 自查：生成前数总数，< 5 条时回头重扫每章的小开门/小关门和连接词

### 严格规范

- source_lines 必须是真实行号（从文件实际读取）
- problem 字段禁止方法论术语（"大开门力度不足/小关门未固化"等）
- options.content 必须写出具体修改文字（大开门要写出实际开场话术，小关门要写出实际过渡句）

## 失败模式

### F1：废话开场漏报（致命）

**特征**：大开门是"大家好，我是XXX，今天给大家分享..."，但未生成 critical suggestion。

**危害**：开场是整场演讲印象最重要的30秒。漏报最严重的结构问题直接让 review-structure 失去价值。

**触发条件检测**：在大开门为废话模式的测试用例中，检查是否有 dimension=`大开门`、severity=`critical` 的 suggestion。

### F2：类型判断越俎代庖

**特征**：ppt-hours.md 未填写分享类型，AI 自行从 slide-content.md 的结构特征推断（如"看起来有故事弧所以是分享型"）并执行对应维度，而不是询问用户。

**危害**：分享型/汇报型决定了维度5/6哪个被执行——推断错误会导致用不适用的框架审查演讲，产生误导性建议（给汇报型报一波三折缺失问题）。

**触发条件检测**：在 ppt-hours.md 分享类型字段为空的测试用例中，检查 AI 是否询问用户而非自行推断并直接执行维度5或6。

### F3：思维模型滥输出

**特征**：匹配度 60-70 分的模型也生成了 suggestion；或同一次审查输出超过 3 条思维模型建议。

**危害**：I5（思维模型）是"锦上添花"的维度，低置信度的模型识别只会引入噪音，让讲者为一个不太吻合的框架耗费时间思考。

**触发条件检测**：检查所有 dimension=`思维模型识别` 的 suggestion，match_score 是否全部 ≥80；总数是否 ≤3。

### F4：小开门/小关门检查遗漏

**特征**：某章明显缺少小开门（直接从论点开始），但未生成任何 suggestion。

**危害**：小开门/小关门是每章的进出仪式感，缺失会让多章节演讲变成平铺流水账，听众无法区分章节边界。

**触发条件检测**：对每章逐一检查，遍历所有章节（不只是第一章和最后章）。

### F5：时长匹配建议无具体依据

**特征**：维度8报告 Slide 偏多，但 options 只说"建议减少 Slide 数量"，没有指出哪几页可合并。

**危害**：D3B 决策要求给出具体建议。没有具体依据的建议让讲者无从下手，形同废话。

**触发条件检测**：维度8的 suggestion options 中是否包含具体的"第N页和第M页内容较简单，建议合并"的说明。

## 评分维度

### D1：开关门质量（25 分）

关联主需 **I1 开关门力度**。

**量化检测**：
- 大开门识别正确（有效钩子 vs 废话模式）= 12 分
- 大关门识别正确（升华+行动号召+可复述 vs 谢谢结尾）= 13 分
- options 写出基于演讲实际内容的具体话术（不是方向说明）= 必要条件

| 分数段 | 描述 |
|--------|------|
| 22-25 | 大开门和大关门均正确识别问题/确认质量；有问题时给出基于演讲实际素材的具体替代话术选项 |
| 15-21 | 识别基本正确，但 options 的 content 缺乏具体性（只有方向说明）|
| 8-14 | 大开门或大关门有漏报（明显废话开场未标 critical）|
| 0-7 | 两者均漏报；或废话开场被判为"有效" |

### D2：章节钩子覆盖率（25 分）

关联主需 **I2 章节钩子**。

**量化检测**：
- 检查所有章节（不只是第一章）的小开门/小关门 = 各章必须遍历
- 每处缺失或废话正确触发 suggestion

| 分数段 | 描述 |
|--------|------|
| 22-25 | 遍历所有章节的小开门/小关门；缺失/废话全部触发 suggestion；options 写出基于本章真实素材的具体钩子话术 |
| 15-21 | 遍历了主要章节，但遗漏 1-2 个章节的检查；或 options 话术与本章素材脱节 |
| 8-14 | 只检查了部分章节（如只检查第一章小开门）|
| 0-7 | 完全未检查章节级小开门/小关门 |

### D3：叙事弧度判断（20 分）

关联次需 **I3 叙事弧度** + 类型判断逻辑。

| 分数段 | 描述 |
|--------|------|
| 17-20 | 正确从 ppt-hours.md 读取分享类型；类型明确时执行对应维度（5 或 6）；类型缺失时询问用户而非自行推断；一波三折/金字塔的判断符合演讲实际结构 |
| 12-16 | 类型读取正确但判断有轻微偏差；或类型缺失时有一定程度的推断（但说明了推断依据）|
| 6-11 | 类型缺失时静默自行推断并执行；或两个维度都执行而无说明 |
| 0-5 | 忽略分享类型，固定执行某一个维度 |

### D4：连接词与格式合规（15 分）

关联次需 **I4 连接流畅** + I5。

| 分数段 | 描述 |
|--------|------|
| 13-15 | 逻辑连接词检查覆盖所有章节过渡；机械切换全部触发；思维模型识别严格遵守 ≥80 门槛和3条上限；source_lines 真实行号；problem 字段无方法论术语 |
| 9-12 | 主要覆盖，但有 1-2 个过渡检查遗漏；或思维模型输出了 match_score 70-79 的建议 |
| 0-8 | 逻辑连接词检查严重不足；或思维模型大量滥输出（> 3 条或 match_score < 70）|

### D5：时长匹配建议具体性（15 分）

关联次需 **I3** + D3B 决策。

| 分数段 | 描述 |
|--------|------|
| 13-15 | 正确计算实际 Slide 数与合理范围；偏差时具体指出哪几页可合并或哪个论点可拆分；severity 对应偏差程度 |
| 9-12 | 检测偏差正确，但 options 只有方向说明（"建议减少 Slide"），无具体页面指向 |
| 0-8 | 时长匹配未检查；或计算公式错误（未使用 ×0.6 ~ ×1.2 范围）|
