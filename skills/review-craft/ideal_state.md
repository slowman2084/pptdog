# 理想态：review-craft

## 概述

`review-craft` 是 pptdog 审查链路中专注于**演讲技巧增益机会**的审查器。与其他审查器检查"问题"不同，这个 Skill 审查的全是**机会**——当前内容已经可用，但可以通过运用演讲技巧变得更有设计感、更有记忆点、更能触发听众行动。

四类技巧：**递进式深挖**（论点从表层到深层逐步揭示）、**Call Back**（回扣前置素材产生精心设计感）、**暗线**（贯穿全场的隐性线索）、**方法论楔子**（给听众可带走的工具和延伸讨论入口）。

所有建议的 severity 最高为 `major`（强烈推荐），不会有 `critical`。

---

## 需求体系

### 固有需求分类（视角 A：演讲技巧完整性）

| ID | 固有需求 | 定义 |
|----|---------|------|
| I1 | 深度机会 | 识别论点停在表层、有更深洞见可以展开的机会 |
| I2 | 回扣设计 | 识别前置素材可被后续回扣的机会，产生"早就设计好"的感觉 |
| I3 | 叙事暗线 | 识别贯穿全场的隐性线索设计机会 |
| I4 | 行动楔子 | 识别关门缺乏可带走工具或延伸讨论入口的机会 |

### 主需与次需

**主要需求：**

- **I1 深度机会** → 落地为：每个核心论点检查是否有递进可能；**必须在 details.md 中找到更深层素材才生成 suggestion（D3A）**；无 details 素材时不报（不推荐讲者编造第2层内容）
- **I2 回扣设计** → 落地为：扫描开门素材（故事/数字/场景）是否在后续被回扣；找到可回扣但未回扣的素材时生成 suggestion；给出具体的回扣话术（不是"建议在关门回扣"这种方向说明）

**次要需求：**

- **I3 叙事暗线** → 任何演讲都尝试识别（D2B）；无天然素材时提炼潜在暗线并给出轻量实现方案；改动成本高的方案标注"较大改动"
- **I4 行动楔子** → 检查关门部分是否有可带走工具和延伸讨论入口；两者均缺时为 major；只缺其中一项时为 minor

### 建议数量约束（D1B）

- **最少 3 条**，有意义才报，不强凑
- 若仅发现 3 条高价值机会，在摘要中说明："本次审查发现 3 条高价值机会，其余维度当前演讲已有较好处理"
- 期望范围：4-8 条（全量模式）
- 不为凑数生成低价值建议：宁可只报 3 条真实机会，也不报 6 条掺杂噪音的建议

```json
{
  "reviewer": "craft",

  "slug": "<slug>",
  "ts": "<ISO timestamp>",
  "suggestions": [
    {
      "id": "craft-001",

      "dimension": "递进 | Call Back | 暗线 | 方法论楔子",
      "severity": "major | minor",
      "type": "opportunity",
      "context": "第N章·论点X.Y「标题」——问题所在部分",
      "source_lines": { "start": N, "end": M },
      "original": "从 slide-content.md 直接引用的原文片段",
      "problem": "口语化描述机会（如：这里可以做个回扣，让听众感觉你早就设计好了）",
      "options": [
        { "label": "A", "description": "方案简述", "content": "具体可直接使用的文字/话术", "recommended": true, "reason": "推荐原因" },
        { "label": "B", "description": "...", "content": "...", "recommended": false, "reason": "" },
        { "label": "C", "description": "保持原样", "content": "（不修改）", "recommended": false, "reason": "" }
      ],
      "recommended": "A"
    }
  ]
}
```

**options.content 写法规范**：
- 递进式：直接写出第2层/第3层的具体论点内容（来自 details.md）
- Call Back：直接写出回扣话术（"还记得我开头说的那次...，现在有了这个方法，..."）
- 暗线：写出每章小关门的暗线推进方式（具体内容，不是描述）
- 方法论楔子：写出 checklist/框架/话术的具体文字
- **禁止**：`"建议深挖这个论点"` / `"可以考虑在关门加入工具"` 等方向说明

## 质量标准

### 维度 1：递进式深挖（D3A — 必须有 details.md 素材）

**识别流程**：
1. 对每个核心论点的演讲口头说，问："如果懂行的听众在台下，他会说'这个我知道了，然后呢？'的时刻在哪里？"
2. 找到停在表层的时刻后，**查 details.md** 对应论点，找更深层素材
3. **有明显更深层素材**（answers "然后呢" 的内容）→ 生成 suggestion
4. **details.md 无更深素材**：不生成 suggestion（不推荐讲者凭空编造）
5. **details.md 不存在**：在状态摘要中标注"details.md 不存在，跳过递进素材检查"，仍可基于 slide-content 的演讲深度判断是否表浅

**options 写法**：
- A（三层递进）：写出第1/2/3层的具体论点内容，标注预计增加页数
- B（两层递进）：写出第1/2层，标注预计增加页数
- C（保持原样）

severity：`major`=核心论点停表层且 details 有明显可展开深层素材 / `minor`=可以更深但当前深度也可接受

### 维度 2：Call Back

**识别流程**：
1. 扫描大开门：提取所有具体素材（故事细节/数字/场景/提问）
2. 扫描后续章节和大关门：检查这些素材是否被回扣
3. 已回扣→不生成；未回扣且有强回扣价值→生成 suggestion

**强回扣价值判定**：开门素材具体（有数字/有场景/有人物）且后续章节的内容与之有呼应关系

**options 写法**：必须写出具体的回扣话术（能直接粘贴到演讲口头说的文字）

severity：`major`=开门故事/数字后续完全未回扣（错失最强设计感） / `minor`=有回扣机会但属于锦上添花

### 维度 3：暗线（D2B — 任何演讲都尝试）

**识别流程**：
1. 扫描全文，寻找潜在暗线素材：贯穿全场的框架/路线图/递进模型/可累积的视觉元素
2. **有天然素材**：生成 suggestion，A 选项为完整暗线设计，B 选项为轻量方式（演讲口头说埋线，不加 Slide）
3. **无天然素材**：尝试从内容共性提炼潜在暗线，生成"轻量暗线"suggestion，标注"这是 AI 的尝试性提炼，改动较大，仅供参考"
4. 实在找不到任何暗线可能性时，可不生成（不强凑）

**severity**：`major`=有天然素材（路线图/框架）但完全未设计暗线 / `minor`=需要较多提炼/改动才能实现

### 维度 4：方法论楔子

**识别流程**：
检查 slide-content.md 的关门部分（最后 1-2 张 Slide）：
- 是否有可带走的工具（checklist / 步骤口诀 / 决策树 / 一页纸框架）
- 是否有延伸讨论入口（"欢迎会后来聊" / 二维码 / 文档链接）

两类都有→不生成；缺一项→minor；两类都没有→major

**options 写法**：A（加工具 Slide，写出具体工具内容） / B（加延伸入口话术，写出具体话术文字） / C（两者都加，推荐）/ D（保持原样）

## 失败模式

### F1：递进 options 缺乏具体内容（最常见）

**特征**：递进式 suggestion 的 options 写"第2层：深挖告警分级的底层逻辑"而不是"第2层：「告警分级不是数量问题，是粒度问题——同一个服务 P1 和 P3 用同一个处理流程，重要信号被噪音淹没」"。

**危害**：讲者看完不知道第2层应该讲什么，建议无法落地。递进式建议的核心价值就是把 details.md 里的深层素材"翻译"成可以直接加入演讲的内容。

**触发条件检测**：options 中 content 字段是否包含具体论点文字（引号内有实质内容），而非方向描述（"可以深挖..."/"建议展开..."）。

### F2：Call Back 话术空洞

**特征**：Call Back suggestion 的 options 写"在关门加入对开门素材的回扣"，但 content 字段写"建议在关门引用开门的故事"，而不是写出实际的回扣话术。

**危害**：讲者不知道回扣话术要怎么说。Call Back 的技巧感来自具体的表达——"还记得我开头说的那 32 分钟吗？现在有了这个监控体系，那个问题 8 分钟就能解决"——这句话要由 AI 写出来，不是让讲者自己想。

**触发条件检测**：Call Back 的 options content 是否包含具体的过去素材引用 + 当前呼应的完整话术结构。

### F3：对没有 details 素材的论点强行生成递进建议

**特征**：details.md 中某论点的内容只有表层数据，AI 仍然生成了三层递进建议，第2/3层内容是 AI 自己编造的"可能的深层洞见"。

**危害**：递进的可信度来自讲者确实有这些素材。AI 编造的第2层内容可能与讲者的实际经验完全不符，讲者在台上讲到第2层时无法自圆其说。

**触发条件检测**：在 details.md 缺少对应深层素材的测试用例中，检查是否仍有递进 suggestion 生成（应为0条）。

### F4：暗线建议改动成本未标注

**特征**：暗线 suggestion 的 A 选项要求"每章小关门各新增1页暗线进度 Slide"，但未标注改动成本（新增页数/对现有结构的影响）。

**危害**：讲者在决策时无法判断这个改动值不值得。暗线是改动成本较高的技巧，讲者需要知道"我采纳 A 选项意味着新增 N 页，调整 M 处小关门"才能做出理性决策。

**触发条件检测**：暗线 suggestion 的 options A 中是否标注预计新增页数和需要修改的位置数量。

### F5：方法论楔子 options 未基于演讲实际内容

**特征**：方法论楔子 suggestion 的工具 Slide 内容是通用框架（"加一个 3 步方法论 Slide"），而不是基于演讲中已有的方法论整理出来的具体 checklist/口诀。

**危害**：通用框架不如从演讲本身提炼的工具有价值——讲者讲了 45 分钟，最后给听众的"可带走工具"应该是这 45 分钟内容的精华提炼，而不是一个独立的通用模板。

**触发条件检测**：方法论楔子的工具 Slide 内容是否引用了演讲中实际出现过的具体步骤/数字/判断标准。

## 评分维度

### D1：递进识别质量（25 分）

关联主需 **I1 深度机会**。

**量化检测**：
- 有 details 深层素材的论点被正确识别 → 每个 +5 分
- 无 details 素材的论点未生成递进建议（D3A 守门）→ +5 分加成
- options content 写出具体第2层内容（非方向说明）→ 必要条件

| 分数段 | 描述 |
|--------|------|
| 22-25 | 正确识别有 details 深层素材的论点；未对无素材论点生成递进；options 写出基于 details.md 真实素材的具体第2/3层内容 |
| 15-21 | 识别基本正确，但 options 的 content 缺乏具体性（方向说明代替实质内容）|
| 8-14 | 对无 details 素材的论点也生成了递进建议（AI 编造）|
| 0-7 | 未执行递进识别；或所有建议都是通用框架提示 |

### D2：Call Back 识别与话术质量（25 分）

关联主需 **I2 回扣设计**。

**量化检测**：
- 开门素材（故事/数字/场景）被正确提取 → 基础
- 未回扣的强价值素材被识别 → 各 +8 分
- options content 写出具体回扣话术（含"还记得...现在..."结构）→ 必要条件

| 分数段 | 描述 |
|--------|------|
| 22-25 | 正确提取开门素材；识别未回扣的强价值素材；options 写出包含前置素材引用 + 当前呼应的完整话术 |
| 15-21 | 识别正确，但回扣话术不完整（只有前半句或只有后半句）|
| 8-14 | 识别了 Call Back 机会但 options 只有方向说明（"建议回扣开门故事"）|
| 0-7 | 未执行 Call Back 扫描；或把已有回扣的素材也标为"未回扣" |

### D3：暗线识别与方案可行性（20 分）

关联次需 **I3 叙事暗线**。

| 分数段 | 描述 |
|--------|------|
| 17-20 | 尝试识别每场演讲的潜在暗线；有天然素材时生成完整暗线方案（含每章推进方式+改动成本标注）；无素材时提炼轻量暗线并标注"尝试性提炼，改动较大" |
| 12-16 | 有天然素材时正确识别，但改动成本未标注；或无素材时静默不生成（未尝试提炼）|
| 6-11 | 只在极明显的情况才生成（等同于漏报大多数暗线机会）|
| 0-5 | 未执行暗线识别 |

### D4：方法论楔子质量（15 分）

关联次需 **I4 行动楔子**。

| 分数段 | 描述 |
|--------|------|
| 13-15 | 正确检查关门工具和延伸入口；工具 Slide 内容基于演讲中已有的具体方法论（不是通用框架）；话术写出具体文字 |
| 9-12 | 正确识别缺失，但工具内容是通用框架而非基于演讲提炼 |
| 0-8 | 未检查方法论楔子；或两类缺失均未触发 suggestion |

### D5：格式合规与建议数量（15 分）

关联主需 I1+I2 + D1B 最少 3 条。

**量化检测**（4 项）：

| # | 检测项 | 通过标准 |
|---|--------|---------|
| 1 | 建议总数 ≥ 3 条 | 不强凑，但有意义建议不足 3 条需说明原因 |
| 2 | source_lines 真实行号 | 不可估计 |
| 3 | problem 字段无方法论术语 | 不含"递进式识别/Call Back 缺失/暗线机会"等术语 |
| 4 | JSON 格式合规（schema 通过）| 所有必填字段存在，类型正确 |
