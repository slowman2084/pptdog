# review-craft — 演讲技巧机会审查

> **本 skill 由 `/ppt-review` 自动调用，不需要手动触发。**
>
> 本 skill 审查的全是「机会」，不是「错误」。
> severity 最高为 `major`，代表"强烈推荐考虑"；`minor` 代表"有机会但不强求"。

## Preamble

```bash
~/.pptdog/bin/pptdog-update-check 2>/dev/null || true
~/.pptdog/bin/pptdog-learnings-search --limit 3 --format pretty 2>/dev/null || true
```

读取文件：
```bash
SLUG="${PPTDOG_SLUG:-$(ls -t $HOME/.pptdog/projects/ | head -1)}"
SLIDE_CONTENT="$HOME/.pptdog/projects/$SLUG/slide-content.md"
DETAILS="$HOME/.pptdog/projects/$SLUG/details.md"
TS=$(date +%Y%m%d-%H%M%S)
OUT="$HOME/.pptdog/projects/$SLUG/review-suggestions/craft-${TS}.json"
mkdir -p "$(dirname $OUT)"
```

## 审查维度

### 维度 1：递进式深挖机会

**什么是递进：**
讲者先说一个表层观点（大家都知道），然后主动说"我知道这是老生常谈"，
再深挖一层（听众觉得这层才有意思），最后再深挖一层更细节的问题（让听众意外）。
视觉上变成 2-3 张 PPT 逐步递进，每张揭示更深一层的洞见。

**识别信号：**
- 某论点的演讲口头说只停在了表层结论（"我们需要提升效率"）
- details.md 中有更深层的素材，但 slide-content 没有展开
- 某论点可以问 3 个连续的"为什么"且每层答案都不同

**AI 识别逻辑（CoT）：**
对每个核心论点，问自己：
> "如果一个懂行的听众在台下，他会在心里说'这个我知道了，然后呢？'的时刻在哪里？"

找到这个时刻后，看 details.md 里有没有更深层的素材可以展开。

**生成 suggestion 格式：**
```
💡 递进机会：论点「[论点标题]」停在了表层

当前状态：[简述当前只说到了哪一层]
听众内心："这个我知道，然后呢？"

Qc-progressive-N：要把这个论点做成递进式吗？

A. 是，三层递进（变成 3 张 PPT）
   第1层（现在已有）：「[当前论点]」
   第2层（深挖）：「[具体的第二层洞见，来自 details.md 的素材]」
   第3层（再深挖）：「[具体的第三层洞见，最让听众意外的点]」
   → 预计增加 2 张 Slide，口头说共增加约 [N] 字

B. 是，两层递进（变成 2 张 PPT）
   第1层：「[当前论点]」
   第2层（深挖）：「[具体的第二层洞见]」
   → 预计增加 1 张 Slide

C. 保持原样（这个论点点到为止即可）

> 💡 推荐 [A/B/C]：[具体推荐理由，例：这个论点背后有强劲的反常识内容（details.md 第X行），三层递进冲击力最强]
```

**severity 判定：**
- `major`：核心论点停在表层，details.md 里有明显可展开的深层素材
- `minor`：论点可以更深，但当前深度也能接受

### 维度 2：Call Back 机会

**什么是 Call Back：**
在演讲后期，有意回扣前面说过的某个细节/数字/场景，
产生"原来早就埋好了"的惊喜感，让整场演讲有精心设计感。

**AI 识别逻辑：**
扫描 slide-content.md，寻找"可回扣素材"：
- 开门时用的故事细节/数字/场景
- 第1章/第2章出现的某个具体案例名称或数字
- 讲者在前面提出但没有立即回答的问题

然后检查后续章节（尤其是关门）是否已经回扣了这些素材。

**生成 suggestion 格式：**
```
💡 Call Back 机会：「[可回扣的素材描述]」出现在 [位置]，但后续没有被回扣

Qc-callback-N：要在后面回扣这个细节吗？

A. 在大关门回扣：
   → 在关门演讲口头说加入：「[具体的回扣话术，例：还记得我开头说的那次 32 分钟定位故障吗？——现在有了这个框架，那个问题 8 分钟就能搞定。]」

B. 在第X章小关门回扣：
   → 在 [具体位置] 加入：「[具体话术]」

C. 不回扣（保持原样）

> 💡 推荐 A，关门是最强的回扣时机，听众在快结束时记忆会被激活，回扣效果最好。
```

**severity 判定：**
- `major`：开门故事/数字后续完全没有被回扣（错失最强设计感机会）
- `minor`：有回扣机会但属于锦上添花

### 维度 3：暗线机会

**什么是暗线：**
设计一条贯穿全场的视觉/逻辑线索，每讲完一个章节就推进/补全这条线索，
到关门时线索完整呈现，产生"原来全场都在铺垫这个"的惊喜感。

**AI 识别逻辑（CoT）：**

步骤1：判断是否有潜在暗线素材——
- 是否有一个贯穿全场的核心框架/模型/路线图？
- 是否有一个从开门就埋下、到关门才完整的问题/场景？
- 章节之间是否有可以累积的视觉元素？
- 是否可以换一个归纳角度，把现有内容的共性提炼出一条隐性主线？

步骤2：若有潜在暗线，生成 suggestion：

```
💡 暗线机会：AI 发现了一个可能的暗线设计

识别依据：「[说明为什么认为这里有暗线机会，附具体内容依据]」

Qc-darkline-N：要设计这条暗线吗？

A. 是，用「[框架/路线图名称]」作为暗线载体
   → 设计方式：[每章结束时在这张图上做什么，关门时呈现什么]
   → 需要新增的 Slide：每章小关门各新增1页「暗线进度」Slide（共 [N] 页）
   → 视觉要求：每次呈现的图保持一致，只有「已完成/进行中」状态变化
   → 预计增加页数：+[N] 页

B. 更轻量的方式：[描述轻量实现，例：不新增 Slide，只在每章小关门的演讲口头说里加一句暗线推进]

C. 不设计暗线（保持原样）

> 💡 推荐 [A/B/C]：[AI 根据章节数量和素材给出推荐，例：章节数 ≤ 3 且有路线图素材时 A 效果最好；章节数 > 4 时 B 更简洁]
```

步骤3：若无明显暗线机会，不生成 suggestion（不强行推荐）。

**severity 判定：**
- `major`：演讲有天然的路线图/框架素材，非常适合做暗线但完全没有
- `minor`：有暗线机会但需要较多改动

### 维度 4：方法论楔子（延伸讨论入口）

**什么是方法论楔子：**
演讲结束后，听众听得很过瘾，但没有行动，遇到新困难也没有办法沟通。
楔子的作用是：给听众一个「可以直接拿走用」的工具/框架/checklist，
同时埋一个「这里还有更多，欢迎来聊」的入口——实用主义，也打开延伸讨论。

**两类楔子：**
1. **可带走的工具**：checklist / 决策树 / 一页纸框架 / 三步口诀
2. **延伸讨论入口**：「如果你们有类似情况，欢迎会后来聊」/ 二维码 / 文档链接

**AI 识别逻辑：**
检查 slide-content.md 的关门部分和最后几页：
- 有没有给听众一个可以直接拿走用的东西？
- 有没有明确的行动号召（CTA）？
- 有没有延伸讨论的入口？

**生成 suggestion 格式：**
```
问题：演讲结尾没有给听众「可带走的工具」或「延伸讨论入口」

当前关门内容：[简述当前关门在做什么]
风险：听众听完过瘾，但回去以后不知道怎么开始用，没有行动。

Qc-cta-N：要在关门加入方法论楔子吗？

A. 加一页「可带走的工具/框架」Slide
   → Slide 内容建议：[具体的工具/框架/checklist，来自 details.md 的可复用方法论]
   → 演讲口头说：「[具体话术，例：最后给大家一个可以直接带走用的清单——]」

B. 在现有关门 Slide 里加延伸讨论入口
   → 在演讲口头说加入：「[具体话术，例：如果你们团队也在做类似的事，欢迎会后来找我聊——]」

C. 两者都加（工具 Slide + 延伸入口话术）  ← 推荐

D. 保持原样（这场演讲的目的不是让听众行动）

> 💡 推荐 C，工具 Slide 让听众感觉"干货满满"，延伸入口让讲者能持续影响听众。
```

**severity 判定：**
- `major`：分享型演讲没有任何可带走的工具或行动号召
- `minor`：有总结但不够具体/可操作

## JSON 生成规则

### 建议数量约束

> **AI 必须严格遵守：**
> - 每次审查**最少生成 4 条**建议（craft skill 全是机会型，不是问题型）
> - 不允许因为"这场演讲技巧已经很好了"而少于 4 条——机会永远存在
>
> **自查提示（输出前 AI 必须做）：**
> - 每个论点都追问了"这里有没有递进机会"吗？
> - 开门素材有没有被后续章节回扣？
> - 关门有没有给可带走的工具？


> **context 和 problem 字段的写法要求（AI 必须遵守）：**
> - `context` 用一句话说清楚位置：`"第N章·论点X.Y「标题」——[具体位置]"`，让讲者不翻文件就能定位
> - `problem` 必须说人话，禁止用方法论术语堆砌：
>   - ❌ 「隐含假设违规，因果推导链存在逻辑断层」
>   - ✅ 「你说了结论，但没说依据——听众会在心里问：凭什么？」
> - `options` 里每个选项的 `content` 必须是可以直接用的文字，不是方向描述

AI 完成所有维度的审查后，生成以下格式的 JSON，写入 `craft-<ts>.json`：

```json
{
  "reviewer": "craft",
  "reviewer_id": "craft",
  "slug": "<slug>",
  "ts": "<ISO timestamp>",
  "suggestions": [
    {
      "id": "craft-001",
      "reviewer_id": "craft",
      "dimension": "递进 / Call Back / 暗线 / 方法论楔子",
      "severity": "major | minor",
      "source_lines": { "start": 10, "end": 20 },
      "original": "原始内容片段",
      "context": "第N章·论点X.Y「[论点标题]」——[问题发生在哪个部分]",
      "problem": "用一句人话描述机会（例：这里可以做个惊喜回扣，让听众感觉你早就设计好了）",
      "options": [
        { "label": "A", "content": "具体落地方案（可直接复制粘贴的内容）" },
        { "label": "B", "content": "具体落地方案" },
        { "label": "C", "content": "保持原样" }
      ],
      "recommended": "A"
    }
  ]
}
```
