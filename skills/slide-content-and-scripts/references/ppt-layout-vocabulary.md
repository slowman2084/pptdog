# PPT 布局词汇表（供 AI 描述 Slide 内容结构时参考）

> 注意：这些是布局类型的参考词汇，不是可以直接调用的函数。
> AI 在 slide-content.md 的「PPT 上放」字段中，应用这套词汇描述每页 Slide 的视觉结构，
> 让后续使用 gen-slides / pptx skill 的人/工具能够理解意图并实现。

---

## A. 结构页

| 布局名 | 适用场景 |
|--------|---------|
| cover（封面） | 演讲第一页，放标题+演讲者+日期 |
| toc（目录） | 多章节演讲，3-5 章结构时 |
| section_divider（章节分隔） | 每章开始，小开门 Slide |
| closing（封底） | 大关门，放核心金句+联系方式 |
| appendix_title（附录标题） | 补充材料分隔页 |

---

## B. 数据与统计

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| big_number（大数据展示） | 数据型论点，有一个最震撼的核心数字 | 数字要大（占页面 40%+），配一行说明 |
| two_stat / three_stat（双/三指标） | 数据型论点，需要并排展示 2-3 个关键指标 | 每个指标：数字 + 标签 + 趋势箭头（若有） |
| metric_cards（指标卡片） | 汇报型，展示多个 KPI 状态 | 卡片包含：指标名 / 当前值 / 目标值 / 状态 |
| data_table / table_insight（数据表格） | 需要展示多维度数据对比时 | 高亮关键行/列，配一行 insight 说明结论 |
| scorecard / rag_status（评分卡/RAG状态） | 汇报型，多项目/多维度健康状态 | 红/橙/绿 三色状态，一目了然 |

---

## C. 框架与矩阵

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| pyramid（金字塔/阶梯） | 方法论型，有层次递进关系（基础→进阶→高级）| 每层一个关键词，从下到上越来越高阶 |
| process_chevron（流程箭头） | 方法论型，3-5 步线性流程 | 每个箭头块一个动词短语（≤6字）|
| temple（殿堂框架） | 方法论型，有支柱→屋顶结构（N个支撑→1个核心目标）| 适合"X 个措施支撑 1 个目标"的表达 |
| venn（维恩图） | 原理型，两个或三个概念有交集 | 交集区域放核心结论，最多 3 圆 |

---

## D. 比较与评估

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| side_by_side（并列对比） | 对比型，两个方案/做法直接对比 | 左右各一列，维度对齐，胜出项高亮 |
| before_after（前后对比） | 数据型/案例型，改进前 vs 改进后 | 左"改进前"右"改进后"，核心数字大字号，中间加箭头 |
| pros_cons（优劣分析） | 对比型，某个决策/方案的权衡 | 绿色优点列 + 红色缺点列，最后一行写"选择理由" |
| swot（SWOT 分析） | 汇报型，战略/方向评估 | 4象限，每格 2-3 条，不要超过 4 条 |
| metric_comparison（指标对比） | 数据型，多个方案的多维度指标横向比较 | 表格形式，行=方案，列=指标，最优值高亮 |

---

## E. 内容与叙事

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| executive_summary（执行摘要） | 汇报型开头，给高管看的一页全貌 | 问题 / 结论 / 方案 / 影响 四块，每块 ≤2 行 |
| key_takeaway（关键结论） | 章节小关门 / 大关门，固化结论 | 1 个核心结论大字号，下方 2-3 条支撑要点 |
| quote（引用） | 大开门/大关门，用一句有力的话引入 | 引言大字号居中，来源小字右下角 |
| two_column_text / four_column（分栏文字） | 方法论型，并排展示 2-4 个概念/步骤 | 每栏：标题（动词短语）+ 2-3 行说明 |

---

## F. 时间线与流程

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| timeline（时间轴） | 案例型，故事有时间线 / 汇报里程碑 | 每个节点：时间 + 事件名 + 1行说明；高亮关键转折点 |
| vertical_steps（垂直步骤） | 方法论型，步骤有先后依赖关系 | 每步：序号 + 动词标题 + 1-2 行说明 |
| cycle（循环） | 方法论型，流程是闭环（如 Plan-Do-Check-Act）| 3-6 个节点，每个节点一个动词 |
| funnel（漏斗） | 数据型，展示转化/筛选过程 | 每层：标签 + 数量/比率，从大到小 |
| value_chain（价值链） | 方法论型，展示价值流动的各个环节 | 每个链条节点一个核心活动，箭头表示流向 |

---

## G. 团队与特殊

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| case_study（案例研究） | 案例型，完整展示一个案例 | 背景 / 问题 / 方案 / 结果 四格，每格 1-3 行 |
| action_items（行动计划） | 大关门，给听众布置下一步 | 行动项 / 负责人 / 截止日期 三列 |
| checklist（清单） | 方法论型，可操作的检查清单 | 每条：checkbox + 动词短语，≤8条 |

---

## H. 图表（数据可视化）

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| grouped_bar / stacked_bar（柱形图） | 数据型，多维度对比 / 组成结构 | 颜色不超过 4 种；必须有标题和数据来源 |
| horizontal_bar（条形图） | 数据型，需要展示排名/长短对比 | 按大小排序；最长/最短的一条高亮 |
| donut / pie（环形/饼图） | 数据型，展示占比结构 | 不超过 5 个扇区；最重要的扇区高亮并放大 |
| waterfall（瀑布图） | 数据型，展示增量/减量构成 | 起止柱用不同颜色；每柱标注变化值 |
| line_chart（折线图） | 数据型，展示趋势变化 | 高亮关键节点（转折点/目标值）；只保留必要的数据点 |
| pareto（帕累托图） | 数据型，找主要矛盾（20/80 法则）| 柱+线组合；标注 80% 那条线 |
| bubble（气泡图） | 数据型，三维度关系（X轴/Y轴/气泡大小）| 每个气泡标注名称；颜色区分类别 |
| gauge（仪表盘）| 汇报型，展示单指标达成率 | 有目标刻度；当前值大字号；颜色区分健康/预警 |
| stacked_area（堆叠面积图）| 数据型，展示组成结构随时间变化 | 颜色不超过 4 种；标注关键时间点 |

## I. 图片+内容

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| content_right_image（右图文）| 有截图/实物图，左边文字右边图 | 图片要有红框标注聚焦点；文字 ≤4 条 |
| case_study_image（案例+图）| 案例型，有实际截图或效果图 | 图片上方/旁边放案例关键数字 |
| full_width_image（全幅图）| 大开门，用一张有冲击力的图做开场 | 图上叠加一句悬念文字（深色背景+白字）|
| quote_bg_image（引用+背景图）| 大开门/大关门，用图片强化情感 | 图片作为背景，一句话大字号居中 |

---

## J. 仪表盘与特殊

| 布局名 | 适用场景 | 内容要点 |
|--------|---------|---------|
| dashboard_kpi_chart（KPI仪表盘）| 汇报型，一页展示多个 KPI + 图表 | 上方 KPI 卡片，下方对应图表；高亮达标/未达标 |
| stakeholder_map（利益相关方矩阵）| 汇报型，展示各方利益/影响力关系 | 2x2 矩阵，X=影响力，Y=关注度 |
| decision_tree（决策树）| 原理型，展示决策路径和条件 | 从根到叶，每个节点一个判断条件，叶节点是结论 |
| icon_grid（图标网格）| 方法论型，展示 4-9 个并列概念 | 每格：图标 + 标题（动词短语）+ 1行说明 |
| risk_matrix（风险矩阵）| 汇报型，风险管理 | 2x2（概率×影响），每象限放对应风险项 |
| harvey_ball_table（Harvey Ball 评估表）| 汇报型，多项目多维度成熟度对比 | 行=项目，列=维度，单元格=成熟度（空/1/4/半/3/4/满）|

## AI 使用指南

在 `slide-content.md` 的「PPT 上放」字段中，用布局名+内容描述这样写：

```
**PPT 上放：**
布局：before_after（前后对比）
  改进前：故障定位 32 分钟
  改进后：3 分钟（↓90%）
  数据来源：2024 Q3 生产数据
```

```
**PPT 上放：**
布局：process_chevron（流程箭头，4步）
  ① 定义 SLO → ② 覆盖率接入 → ③ 告警分级 → ④ 复盘机制
  高亮：第②步（本章核心）
```

```
**PPT 上放：**
布局：content_right_image（右图文）
  左侧文字：
    - 覆盖率：30% → 99%
    - 定位时长：32min → 3min
  右侧图片：references/monitor-dashboard.png
    红框标注：左上角 SLO 达成率区域
```

这样后续使用 `/gen-slides` 时，PPT skill 可以直接理解意图并选用对应布局。
