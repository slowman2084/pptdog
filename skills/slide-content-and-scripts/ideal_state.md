# 理想态：slide-content-and-scripts

## 概述

`slide-content-and-scripts` 是 pptdog 内容生产链路的终点。它接收 `mindmap.md`（已确认的唯一骨架）和 `details.md`（素材库），输出 `slide-content.md`——每页 Slide 的视觉内容规划 + 对应的演讲口头脚本。

这个 Skill 不是"重新设计演讲"，而是**高效地将已确认的框架翻译成可执行的初稿**。AI 扮演的是**高效执行者**：忠实走完完整流程，生成有实质内容的初稿，让用户拿到后能直接审阅和微调。

---

## 需求体系

### 固有需求分类（视角 B：认知负荷分配）

本 Skill 的表面需求——"把 mindmap 转成每页 PPT 内容"、"PPT 和讲稿不能重复"、"结构不能被改"、"布局要选对"、"时长要可控"——从**认知负荷分配**视角归类为以下固有需求：

| ID | 固有需求 | 定义 |
|----|---------|------|
| I1 | 叙事主权 | 讲者对自己的逻辑链条有完全掌控感，不被 AI 越俎代庖 |
| I2 | 双通道协调 | 视觉通道（PPT）与听觉通道（演讲）的信息分配是核心设计决策 |
| I3 | 细粒度执行 | 从章→节→页→伪动画分组的逐层细化，不跳步 |
| I4 | 专业质感 | 成品不像 AI 模板填充物，有专业演讲稿的品质感 |
| I5 | 容错友好 | 初稿不完美没关系，但问题位置可快速定位 |

### 主需与次需

**主要需求（决定理想态的核心走向）：**

- **I1 叙事主权** → 落地为：`mindmap.md` 是唯一骨架，绝对不可重组/合并/拆分/新增章节；所有结构性决策权归用户
- **I2 双通道协调** → 落地为：空姐效应是灵魂约束；每页必须明确区分"PPT负责什么 / 演讲负责什么"；视听重叠度 <30% 是硬指标

**次要需求（影响质量标准权重和细节规则）：**

- **I3 细粒度执行** → 5 步流程（验证→时长→开篇→逐章规划→收尾）每步有明确产出，不可跳过
- **I4 专业质感** → 布局词汇精准匹配论点类型、演讲脚本口语化程度达标、开门关门有力度
- **I5 容错友好** → 全局自检步骤集中汇报所有问题，缺素材位置标注 `[需补充]`

### 需求间关系

```
I1(叙事主权) ──┐
                ├── 共同支撑 → 高质量 slide-content.md 初稿
I2(双通道协调)─┘

I3(细粒度执行) ←── 为 I1/I2 提供过程保障（逐层细化确保不遗漏）
I4(专业质感) ←── 为 I2 提供品质保障（视听分工的最终呈现质量）
I5(容错友好) ←── 为 I1 提供安全网（用户始终能定位并修正任何问题）
```

## 输出规格

### 文件路径

`~/.pptdog/projects/<slug>/slide-content.md`

### 文件头元数据

```markdown
# Slide 内容规划 — <演讲标题>

> 生成时间：<ISO 时间>
> 演讲类型：<分享型 | 汇报型>
> 目标时长：<X> 分钟 | 预计实际时长：<Y> 分钟
> 超时状态：<✅ 在预算内 / 🟡 轻微超时 / 🔴 严重超时>
> Slide 总数：<N> 页（含大开门/大关门/各章小开门/各章小关门）

> 图片清单：
> - references/xxx.png → Slide N（论点X.Y）[用途说明]
> - [占位] Slide K → 建议提供 [类型] 截图放到 references/

> ⚠️ 本文件包含 <N> 处 AI 推断/自动修补内容，请逐一确认。（仅当有时出现）
```

### 整体结构

```
大开门（1页）
→ 第1章小开门（1页）→ 论点Slides（N页）→ 第1章小关门（1页）
→ 第2章小开门（1页）→ 论点Slides（N页）→ 第2章小关门（1页）
→ ...（按 mindmap 章节数循环）
→ 大关门（1页）
```

### 单页 Slide 格式（非伪动画）

```markdown
## Slide N — [标题（=论点，≤21字，含数字/痛点）]

**论点类型：** <数据型/案例型/方法论型/原理型/架构型/对比型/状态汇报型>

**PPT 上放：**
布局：<布局名>（<布局中文名>）
[具体内容——数字/步骤/对比项，不是描述说明]

**参考素材（若有）：**
- 图片类：references/文件名，红框标注 [XXX]，说明 [YYY]
- 文字/数据类：[素材：references/文件名，用于：具体用途]

**演讲口头说：**
- 承接上一页：[过渡句，口语化逻辑连接词]
- 核心内容：[直接引用 details.md 的实际证据/案例/Why层]
- 铺垫下一页：[引向下一论点的连接句]
预计时长：约 X 分钟 [可压缩]（如可压缩加标记）

**配合说明：**
[PPT负责：视觉锚点/数字冲击]
[演讲负责：Why + 故事 + 授人以渔]

**素材来源：** （来源：details.md §论点X.Y）
```

[只放第1页内容]

**演讲口头说：**
[第1页停留内容]
[切换触发词：「...？」]
预计时长：约 X 秒（切换前）

---

## Slide N-b — [揭晓/对比版标题，≤21字]
🎭 伪动画第2页（共M页）｜技巧：[技巧名称]

**PPT 上放：**
布局：[与 N-a 相同布局]
[第1页内容保留（灰色/小字），追加新内容高亮]
⚠️ 意图：利用对比/揭晓制造落差感

**演讲口头说：**
[切换后继续说的内容]

### 大开门/大关门/小开门/小关门格式

各有专属 Markdown 二级标题和固定字段，详见 SKILL.md 原文定义。

### 关于 `[需补充]` 标注

对于 details.md 中确实缺失、AI 无法合理推断的内容，使用以下标注：

```markdown
[需补充：具体缺失什么，建议提供什么类型的内容]
```

示例：
- `[需补充：此论点的具体数据或案例，建议从项目周报/监控面板截取]`
- `[需补充：听众可能质疑的反面论点及你的回应]`
- `[需补充：此处适合放一张架构图或流程截图]`

**原则**：`[需补充]` 仅用于"确实没有素材且无法合理推断"的情况，不用于"AI 偷懒"。能用 details.md 推断出的内容一律填充，并用 `[AI推断，请确认]` 标注。

## 质量标准

### 约束优先级（全部严格执行时的冲突解决顺序）

当多条约束产生冲突时，按以下优先级裁定：

| 优先级 | 约束 | 说明 |
|--------|------|------|
| P0 | mindmap 结构不变 | 绝对不可违反 |
| P1 | 空姐效应（视听分离） | 核心设计原则，仅在有强理由时可弹性处理 |
| P2 | 布局词汇匹配论点类型 | 严格对应 60+ 布局词汇表 |
| P3 | 时长控制 | 超时 >25% 必须在自检中高亮 |
| P4 | 演讲口语化 | 主语/连接词/背稿风险检查 |

### Step 0：内容缺口校验

- 生成完整的 `mindmap ↔ details` 对比表，每个论点一行，状态为 ✅/⚠️/❌
- **决策方式（D3C 自动修补+draft+确认）**：
  - ✅ 完整匹配：直接进入下一步
  - ⚠️ 部分匹配：AI 基于上下文自动补全，在输出中用 `[AI推断，请确认]` 标注
  - ❌ 完全缺失：AI 基于相邻论点和章节主题给出最合理推断，同样标注 `[AI推断，请确认]`
- 不中断流程等待回答，但所有推断/修补点在文档中明确标记
- 在 Step 5 全局自检阶段统一汇总所有推断/修补点供用户批量确认

### Step 1：时长估算

- 按内容量而非页数估算：
  - 纯数据型：约 1 分钟/页
  - 完整案例型：约 2.5-3.5 分钟/页
  - 深度案例+Why层：约 3.5-5 分钟/页
- 伪动画多页整组按一个论点估算，不按页数叠加
- 打印完整时长汇总表（每行：论点名 / 内容量等级 / 伪动画标记 / 可压缩标记 / 预估分钟）
- **超时 >25%**：在汇总表中用 🔴 标记，并在 Step 5 自检中作为第一项汇报

### Step 2：大开门

- 必须基于 details.md 中找到的实际素材生成候选
- 3 个候选方案必须有类型差异（好故事 / 反常识 / 问题引导），每个写出实际开场文字
- AI 给出推荐理由（基于素材强度："你的 details 里那个 XX 案例细节丰富，冲击力最强"）
- **决策方式（D3C）**：AI 直接选定推荐方案生成完整 Slide（含 PPT 内容 + 演讲脚本），在 Slide 末尾附注：
  ```
  💡 大开门备选方案 B/C 见附录 A.1（如想切换可直接替换）
  ```

### Step 3：逐章节主体

- **P0 硬约束**：严格按 mindmap.md 章节顺序，不跳章节、不重排、不合并、不拆分
- 每章依次执行：小开门 → 各论点 Slides → 小关门
- **伪动画识别**：AI 内部判断是否命中场景（悬念追加/逐步揭示/对比展开/因果链/数据冲击）
  - 命中：直接生成伪动画组（不询问，因为 D1B 是高效执行者模式），在组首附注命中理由
  - 未命中：直接生成单页
- **PPT 内容要求**：必须写出"布局名 + 具体内容"（数字/步骤/对比项），禁止描述性文字
- **演讲脚本要求**：
  - 主语必须是"我"，出现"我们"时自动改写并在全局自检中列出
  - 禁止占位符（`[具体内容]`/`[演讲者填写]`等）
  - 允许 `[需补充]`（仅限确实无素材情况）和 `[AI推断，请确认]`
- **空姐效应（P1）**：每页必须包含"配合说明"块，明确 PPT 负责 vs 演讲负责

### Step 4：大关门

- 与大开门形成首尾呼应（好故事开门→升华关门 / 问题引导→Playbook / 反常识→数据全景）
- 候选方案基于所有章节实际素材，写出实际关门文字
- **决策方式（D3C）**：AI 直接选定最佳方案生成完整 Slide，附注备选

### Step 5：全局自检（容错友好的集中汇报）

按顺序执行以下 5 项检查，每项产出结构化报告：

1. **时长最终核算**：每页一行（Slide编号 / 论点 / 预估时长 / 可压缩标记）；超时时列出可压缩项
2. **伪动画连续性**：每组检查（布局一致 / 视觉连续 / 切换触发词存在 / 时长合理）；不一致则标注
3. **空姐效应扫描**：逐页检查 PPT 是否有完整句子、视听重叠度估算；超标页列出
4. **标题论点化检查**：话题标题 vs 论点标题分类，需修改的列表
5. **主语 & 口语化**："我们"出现位置列表 + 改写建议；背稿风险句列表

**关键产出：AI 推断/修补/需补充汇总清单**

```markdown
## 📋 待确认事项汇总

### AI 推断内容（<N> 处）
| 位置 | 原始缺口 | AI 推断内容 | 置信度 |
|------|---------|-----------|--------|
| Slide 3 | details 缺少具体数据 | "根据项目经验，转化率提升约 30%" | 中 |

### 需补充内容（<N> 处）
| 位置 | 缺失类型 | 建议提供 |
|------|---------|---------|
| Slide 7 | 数据证据 | 从监控面板截取 Q4 转化率趋势图 |

### 自动修补记录（<N> 处）
| 位置 | 原始矛盾 | 修补方案 |
|------|---------|---------|
| §2.3 | mindmap 说"A导致B"但 details 写"B由C引起" | 以 details 为准，已调整 Slide 9 的因果表述 |
```

### Step 6：保存

- 写入 slide-content.md，含完整文件头元数据
- 打印统计摘要：总页数 / 大开关门数 / 小开关门数 / 论点 Slide 数 / 伪动画组数
- 打印预计总时长和超时状态
- 打印 `[需补充]` / `[AI推断]` 计数
- 推荐下一步：`/ppt-review` → `/gen-slides`

## 失败模式

### F1：骨架篡改（致命）

**特征**：Slide 顺序与 mindmap.md 不一致；AI 合并/拆分/新增了章节；AI "觉得更合理"重排了论点顺序。

**危害**：讲者认不出"这是我设计的结构"，无法自然地讲述。mindmap 是人的决策结果，不是 AI 的输入素材。一旦骨架被篡改，后续所有内容（即使质量很高）都建立在错误的基础上。

**触发条件检测**：生成的 Slide 编号序列 vs mindmap 章节树做结构 diff，任何节点的新增/删除/移动都算命中。

### F2：空姐效应失效（严重）

**特征**：PPT 上放了完整句子；演讲口头说复述了 PPT 内容；单页 PPT 信息点超过 3 个且无拆页建议；"配合说明"块缺失或写成了"PPT和演讲都说同一个意思"。

**危害**：听众不知道该看 PPT 还是听讲者，注意力分散。讲者变成"读 PPT"，失去眼神交流和临场感。这是整个 pptdog 流程中最核心的质量底线之一。

**触发条件检测**：逐页扫描 PPT 内容段，检测完整句子（主语+谓语+宾语完整）；对比 PPT 内容词集与演讲脚本词集，Jaccard 相似度 >50% 即命中。

### F3：占位符偷懒（严重）

**特征**：演讲脚本中出现 `[演讲者填写]`、`[具体内容]`、`[此处插入案例]`；PPT 内容只有关键词堆叠而无实际数字/步骤/对比项；大开门候选是方向说明而非实际文字。

**危害**：讲者拿到后无法使用，等于白跑一趟全流程。占位符 = AI 放弃思考的信号，意味着后续内容可信度为零。

**触发条件检测**：正则扫描 `[.*?]|<待填充>|TODO|FIXME` 类标记；PPT 内容段落平均长度 <10 个字符即疑似偷懒。

### F4：流程跳步（中等）

**特征**：跳过 Step 0 直接开始；跳过 Step 5 全局自检直接保存；某章缺少小开门或小关门；时长从未被估算。

**危害**：遗漏关键质量控制环节。Step 0 跳过意味着可能在错误的基础上生成全部内容；Step 5 跳过意味着问题只能到 `/ppt-review` 才被发现，效率降低。

**触发条件检测**：检查输出文档是否包含每个 Step 的标志性产物（对比表 / 时长汇总表 / 自检报告）。

### F5：伪动画误判（中等）

**特征**：对不适合的论点强制做了伪动画（如纯数据展示页做成悬念揭示）；或对明显适合的论点（如因果链推理）没做伪动画；伪动画组的两页布局不一致。

**危害**：不适合的伪动画让听众困惑（"这有什么好悬念的？"）；错过的伪动画浪费了增强叙事节奏的机会。

**触发条件检测**：对照 SKILL.md 中的伪动画检测表，检查命中/未命中的合理性。

### F6："我们"泛滥（轻微但影响 IP）

**特征**：演讲脚本中大量"我们做了"、"我们发现"、"我们的成果"；全局自检未识别或未改写。

**危害**：演讲变成匿名团队汇报。在外部大会场合，听众来听的是讲者的个人故事和判断。"我们"抹杀了个人 IP 价值。

**触发条件检测**：正则统计"我们"出现频次，>3 次且无必要（非引用原话）即命中。

## 评分维度

### D1：mindmap 骨架忠实度（25 分）

关联主需 **I1 叙事主权**。这是本 Skill 的最高优先级约束（P0）。

| 分数段 | 描述 |
|--------|------|
| 23-25 | Slide 顺序与 mindmap.md 完全一致（逐节点比对通过）；无章节合并/拆分/新增/删除；论点顺序零调整；遍历规则严格执行 |
| 17-22 | 骨架基本遵循，但有 1 处未经确认的微调（如同级论点顺序微调），不影响整体结构 |
| 12-16 | 有 1-2 处明显的结构偏离（如某章论点被合并），但主要章节结构保持 |
| 0-11 | 出现章节级别的新增/删除/合并/重排；或 mindmap 被当作"参考"而非"唯一骨架" |

### D2：空姐效应执行质量（25 分）

关联主需 **I2 双通道协调**。这是本 Skill 的灵魂约束（P1）。

| 分数段 | 描述 |
|--------|------|
| 22-25 | 每页 PPT 仅含关键词/数字/图示/短短语，无完整句子；每页都有明确的"配合说明"（PPT负责vs演讲负责）；视听重叠度 <30%；布局与论点类型严格对应 |
| 17-21 | ≥90% 页面执行达标，有 1-3 页 PPT 内容偏详细或与演讲有轻度重叠（30-50%） |
| 12-16 | 约 70% 页面达标，有多页存在较明显的重叠或 PPT 过于文字化 |
| 0-11 | 多页 PPT 是完整句子；大量页面无"配合说明"块；重叠度普遍 >50% |

### D3：内容充实度与推断质量（20 分）

关联次需 **I3 细粒度执行** + **I5 容错友好**。反映 D4B（骨架+标注待补）的执行质量——核心内容应该充分填充，缺失部分正确标注。

**量化检测标准：**
- 每页 PPT 内容段 ≥ 12 个中文字符 **或** 包含 ≥ 1 个数字/数据点/步骤编号 → 算"充实"
- 每页演讲脚本核心内容段 ≥ 20 个中文字符 → 算"有实质"
- `[需补充]` 出现率 ≤ 10%（相对总论点数）→ 算"标注合理"
- `[AI推断]` 必须附带置信度说明 → 算"标注合规"
- 占位符（`[演讲者填写]`/`[具体内容]`/TODO/FIXME）= 0 → 算"无偷懒"

| 分数段 | 描述 | 检测依据 |
|--------|------|---------|
| 18-20 | 所有论点 Slide PPT 段和演讲段均通过上述检测；无占位符偷懒；`[需补充]` ≤10% 且每条有清晰建议；`[AI推断]` 全部带置信度 | 全部量化指标达标 |
| 13-17 | 核心论点（每章前 2 个）全部通过检测；次要论点有 1-2 处 PPT 段 <12 字符或演讲段 <20 字符；`[需补充]` 在 10-20% | 核心论点全达标，次要论点有 ≤2 处不达标 |
| 8-12 | 有 3-5 处本可以通过检测却使用了 `[需补充]` 或 `[AI推断]`（即 details.md 有素材但 AI 未使用）；或 `[需补充]` >20% | 可填充未填充 + 需补充过多 |
| 0-7 | 占位符（偷懒类）≥3 处；或 ≥30% 的论点 Slide 未通过充实度检测；或 `[AI推断]` 无置信度说明 | 多项量化指标严重不达标 |

### D4：流程完整性与自检质量（15 分）

关联次需 **I3 细粒度执行** + **I5 容错友好**。D1B（高效执行者）要求流程完整走完，不自检等于没做。

| 分数段 | 描述 |
|--------|------|
| 13-15 | 6 个 Step 全部执行，每步有明确产出；Step 5 自检 5 项全部覆盖，产出结构化的待确认清单（推断/需补充/修补三张表）；时长核算到每页 |
| 10-12 | 主要步骤都执行了，但 Step 5 自检有 1-2 项流于形式或缺失 |
| 6-9 | 跳过 1 个 Step（通常是 Step 0 或 Step 5）；或自检报告过于简略（"一切正常"无细节） |
| 0-5 | 多个 Step 被跳过；或完全没有自检环节 |

### D5：开门/关门与专业质感（15 分）

关联次需 **I4 专业质感**。拆分为 3 个可检测子项：

#### D5.1：开门/关门实际内容质量（5 分）

| 分数 | 标准 |
|------|------|
| 5 | 大开门包含 ≥1 个实际细节（时间/地点/人物/数字/情绪描写）；大关门包含 ≥1 个具体行动号召或升华句；每章小开门/关门是完整句子（非占位符） |
| 3-4 | 大开门/大关门有实际内容但缺少具体细节（如"用一个故事开场"但没写出故事）；或小开门/关门有 1-2 个占位符 |
| 0-2 | 大开门或大关门是方向性描述（如"建议用反常识开门"）；或 ≥50% 小开门/关门为占位符 |

#### D5.2：推荐理由的素材引用度（5 分）

| 分数 | 标准 |
|------|------|
| 5 | 推荐理由明确引用了 details.md 中的具体素材（如"details 中 §2.1 的服务器宕机案例有时间线和影响范围，适合做悬念开门"） |
| 3-4 | 推荐理由有指向性但不够具体（如"你的素材里有好几个故事"但未指明哪个） |
| 0-2 | 推荐理由是泛化陈述（如"故事开门通常效果好"）；或无推荐理由 |

#### D5.3：首尾呼应逻辑自洽性（5 分）

| 分数 | 标准 |
|------|------|
| 5 | 大开门类型与大关门类型形成明确呼应关系（问题引导→Playbook / 故事开门→升华 / 反常识→数据全景），且在文档中注明了呼应逻辑 |
| 3-4 | 有呼应意识但逻辑连接较弱（如都用了故事但两个故事无关） |
| 0-2 | 无首尾呼应；或大开门和大关门风格完全割裂 |
