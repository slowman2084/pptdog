## Step 1：预计时长估算与超时预警

在生成任何 Slide 之前，先做时长估算：

**估算规则（AI 内置）：**
- 开门：约 1.5 分钟
- 关门：约 1 分钟
- 每个核心论点：根据 details.md 内容量估算（轻量论点 1.5 分钟，有案例论点 3 分钟，复杂论点 4 分钟）
- 章节过渡 Slide：约 0.5 分钟

**打印时长汇总表：**

```
论点             | 预计时长 | 备注
----------------|---------|----
开门             | 1.5 min |
1.1 <论点标题>   | 3 min   | 有案例
1.2 <论点标题>   | 2 min   | 纯数据
...
关门             | 1 min   |
----------------|---------|----
合计             | XX min  | 目标：<Y> 分钟
```

**超时提示：**
- 超目标 ≤10%：提示"略有超时，可轻微压缩"
- 超目标 10%-25%：🟡 警告，建议标注 [可压缩] 论点
- 超目标 >25%：🔴 严重超时，建议删减章节或合并论点

如有超时，**在继续前询问用户：**

**Qs-3：预计时长 {XX} 分钟，超出目标 {YY%}，怎么处理？**

```
A. 先生成完整版，生成后再决定删什么          ← 推荐，先看全貌
B. 现在就标出可压缩的论点，确认后再生成
C. 删掉最后一个章节（一般可省 20-30% 时间）
D. 自定义：我想删/合并某个具体章节（告诉 AI）
```

> 💡 超时 ≤15% 时选 A 通常没问题，现场可以语速稍快；超时 >25% 建议选 B 或 C，否则现场压力太大。

## Step 2：开门 Slide 规划（必须包含）

> **开门目标：前 30 秒，让听众放下手机。**
> 判断听众是谁，选择最合适的开门类型。

**AI 根据 ppt-hours.json 和 mindmap.md 的主题，给出开门类型建议：**

```
类型 A：好故事
→ 用一个真实的、有细节的、有情绪的故事开场
→ 故事要和主题直接相关，但不要太早揭示结论
→ 适合：有真实案例的分享型演讲

类型 B：问题引导
→ 提出一个听众一定遇到过的问题/困境
→ 用反问激发思考："你有没有遇到过……"
→ 适合：解决方案型分享，听众痛点明确

类型 C：反常识
→ 提出一个与听众预期相反的数据或结论
→ 制造认知冲突，引发好奇
→ 适合：有数据支撑的技术/产品分享
```

**Qs-1：开门方式用哪个？**

```
A. 好故事 — 用真实的、有细节的故事开场，不提前揭示结论
   适合：有真实案例的分享型演讲
B. 问题引导 — 提出听众一定遇到过的困境，用反问激发思考     ← 分享型推荐
   适合：解决方案型分享，听众痛点明确
C. 反常识 — 提出一个与听众预期相反的数据或结论，制造认知冲突
   适合：有数据支撑的技术/产品分享                          ← 汇报型推荐
D. 自定义：我有自己的开场方式（描述给 AI）
```

> 💡 判断技巧：你有没有一个"让听众说'哇，这我没想到'"的数字或故事？有 → 选 C 或 A；没有 → 选 B 最稳。

**AI 给出推荐后，用户确认类型，AI 生成：**

```markdown
## 开门 Slide（前30秒目标：让听众放下手机）

**PPT 上放：**
[一句话 / 一个数字 / 一个问题——引发好奇，不超过20字]
[不放答案，不放结论，放"悬念"]

**演讲口头说：**
[完整的开场故事/问题/反常识内容，口语化]
[要有细节：时间、地点、人物、情绪]
[结尾引向"所以今天我想和大家分享……"]
预计时长：约 1.5 分钟

**配合说明：**
PPT 上只有一个钩子，听众的眼睛还没有被 PPT 吸引走，
演讲者用声音和故事控制全场注意力。

**开门类型：** <好故事 / 问题引导 / 反常识>
**选择理由：** <AI 说明为什么推荐此类型>
```

## Step 3：主体 Slide 逐页规划

> **对每一个章节和论点，生成完整的 Slide 规划。**
> 遵循以下设计原则（AI 在生成时严格执行）：

**设计原则（内置，每页 Slide 生成时自动检查）：**

1. **空姐效应**：PPT 上不写你要读的内容。PPT 是视觉锚点，演讲是声音内容，两者互补不重复。
2. **架构图必须有红框/标注**：每张架构图都要注明"红框标注 XXX，说明 YYY"，强迫讲者明确"我要说这个图的哪个部分"。
3. **逻辑连接词口语化**：过渡句使用口语（"更进一步看" / "你有没有遇到过这种情况" / "这背后的原因是"），而不是书面语（"综上所述" / "接下来"）。
4. **每页最多3个信息点**：超过3个就拆成多页，或归并为上一层的标题。
5. **标题=论点**：检查每个标题——是话题（"监控建设"）还是观点（"监控覆盖率从30%到99%，故障定位时长降低90%"）？

**每个 Slide 的生成格式：**

```markdown
## Slide N — [标题（=论点，≤21字，含数字/痛点）]

**PPT 上放：**
- [核心关键词 / 数字 / 对比图说明]
- [若有架构图：注明"红框标注：XXX 模块，说明 YYY 是瓶颈所在"]
- [若有表格/图表：注明数据来源和核心结论]
- ⚠️ 注意：放的是视觉锚点，不是你要读的句子

**演讲口头说：**
- 承接上一页：[过渡句，逻辑连接词，口语化]
- 核心内容：[口语化展开，含故事/案例/数据解读，有 Why]
  - [案例细节，有情景感]
  - [数据解读，"这个数字意味着什么"]
  - [方法论，听众能复用的部分]
- 铺垫下一页：[引向下一个论点的连接句]
预计时长：约 X 分钟 [可压缩]（如可压缩则加标记）

**配合说明：**
[PPT 负责：视觉锚点/数字冲击/架构全局观]
[演讲负责：Why + 情感共鸣 + 授人以渔]
```

**章节过渡 Slide 格式（每章开始时插入）：**

```markdown
## Slide N — 章节过渡：[章节标题]

**PPT 上放：**
- 本章核心问题（一句话）
- 本章将覆盖的论点（3条以内的 bullet）

**演讲口头说：**
- 承接上一章：[总结上一章核心，用一句话]
- 本章预告：[用听众听懂的语言说"接下来我们看……"]
预计时长：约 0.5 分钟
```

## Step 4：关门 Slide 规划（必须包含）

> **关门目标：听众能在 3 分钟内向同事复述核心内容。**

**AI 根据主题和演讲类型，给出关门类型建议：**

```
类型 A：重复（Summary）
→ 把核心3个论点重新说一遍，加强记忆
→ 适合：干货多、听众需要记住操作步骤的分享

类型 B：升华（Elevation）
→ 从技术/案例升华到更大的价值观或方法论
→ "这不只是关于 XXX，更是关于 YYY"
→ 适合：思想性强的分享，有方法论迁移价值

类型 C：流程图（Playbook）
→ 把整个演讲内容整理成一张可操作的流程/清单
→ 听众回去就能照做
→ 适合：How-to 型分享，授人以渔

类型 D：地图（Map）
→ 展示一张"全景图"，显示所有内容的关系
→ 让听众看到整体，理解各部分如何连接
→ 适合：系统架构型、方法论框架型演讲
```

**Qs-2：关门方式用哪个？**

```
A. 重复（Summary）— 把核心 3 个论点重新说一遍，加强记忆
   适合：干货多、需要记住操作步骤的分享
B. 升华（Elevation）— 从技术/案例升华到更大的价值观或方法论    ← 分享型推荐
   适合：思想性强的分享，有方法论迁移价值
C. 流程图（Playbook）— 把演讲内容整理成可操作的流程/清单
   适合：How-to 型分享，听众回去可以直接照做
D. 地图（Map）— 展示一张"全景图"，显示所有内容的关系
   适合：系统架构型、方法论框架型演讲                          ← 汇报型推荐
E. 自定义：我有自己的收尾方式（描述给 AI）
```

> 💡 配合开门：好故事开门 → 升华关门（首尾呼应）；反常识开门 → 重复关门（再次强化）；解决方案型 → 流程图关门。

**AI 给出推荐后，用户确认类型，AI 生成：**

```markdown
## 关门 Slide（目标：听众能3分钟内向同事复述核心内容）

**PPT 上放：**
[核心脉络回顾 / 3个关键词 / 流程图 / 全景地图]
[每个核心论点浓缩为一句话或一个数字]

**演讲口头说：**
- 总结核心：[用3句话回顾3个核心论点]
- 升华主题：[从具体案例升华到可迁移的方法论]
- 授人以渔：["你回去之后，可以用这个方法做……"]
- 结束语：[有力量的收尾，不是"谢谢大家"]
预计时长：约 1 分钟

**配合说明：**
PPT 给听众一张"记忆地图"，演讲者给听众"带走的礼物"（方法/启发）

**关门类型：** <重复 / 升华 / 流程图 / 地图>
**选择理由：** <AI 说明为什么推荐此类型>
```


如超时，列出所有 [可压缩] Slide，询问用户是否压缩或删减。

### 5-B：标题论点检查

遍历所有 Slide 标题，检查：
- 是话题（❌ "监控建设"）还是论点（✅ "监控覆盖率提升至99%，故障定位提速90%"）？
- 有没有废话标题（❌ "发现了一个问题" / ❌ "通过XX策略，突出优势"）？

打印需要修改的标题列表，询问用户确认后更新。

### 5-C：空姐效应最终扫描

检查每页 PPT 的"PPT 上放"部分：
- 有没有完整句子（应该是关键词/数字/图示）？
- 有没有和演讲口头说的内容重叠超过50%？

打印有问题的 Slide 编号，建议简化方式。

### 5-D：逻辑连接词检查

检查每页的"承接上一页"过渡句：
- 有没有 Slide 之间逻辑跳跃（前后无连接）？
- 过渡句是否口语化？

## 附录：设计原则速查

### 空姐效应（核心原则）
> 眼睛盯着 PPT 就离不开了。PPT 上写了的，演讲时就不要再读。

- PPT 是视觉锚点：关键词、数字、对比图
- 演讲是声音内容：Why、故事、情感共鸣、方法论

### 标题=论点（电梯法则）
> 听众在电梯里看到这个标题，3秒内应该能明白你在说什么重要的事。

```
❌ 话题型标题：
  "监控建设"
  "质量体系优化"
  "K8S 不满足我们的诉求"

✅ 论点型标题：
  "监控覆盖率从30%到99%，故障定位时长降低90%"
  "三个措施将P99延迟从2s降至200ms"
  "容器化后资源复用率提升60%，省下XX台服务器"
```

### 开门/关门对应关系
| 演讲特征 | 推荐开门 | 推荐关门 |
|---------|---------|---------|
| 有真实故事、情绪弧度 | 好故事 | 升华 |
| 听众有明确痛点 | 问题引导 | 流程图（给解决方案） |
| 有反直觉的数据 | 反常识 | 重复（加深印象） |
| 系统性框架/全景 | 问题引导 | 地图 |
| 汇报型、结果导向 | 反常识/数字冲击 | 重复 |

### 每页Slide信息量控制
```
1个信息点 = 1个关键词 / 1个数字 / 1张图
每页最多3个信息点

若内容超过3点：
→ 检查是否可以拆成2页
→ 检查是否可以归并为1个上层概念
→ 不要"硬塞"：多即是少
```

### 逻辑连接词口语化示例
```
❌ 书面语：综上所述 / 由此可见 / 接下来 / 此外
✅ 口语化：
  "更进一步看，我们发现……"
  "你有没有遇到过这种情况……"
  "这背后的真正原因是……"
  "但这只是第一层，更深的是……"
  "所以问题来了……"
  "带着这个结论，我们再看……"
```
